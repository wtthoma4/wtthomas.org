Opencode + MapleAI

Better autocomplete, sans Minority Report.

Posted Feb 7, 2026 Updated Feb 9, 2026
By Taylor Thomas
5 min read
Opencode + MapleAI
Opencode + MapleAI

I’ve been playing with Maple, but if an AI tool wants to compete with frontier companies for software development, it must offer a way to avoid the copy-paste game. Keeping AI context up-to-date while I code is an annoying time sink. It’s easier to keep the architecture in my head and tell a bot to re-read a file than to keep every file open in the chat, copying and pasting back and forth.

Opencode is an open source coding agent with harnesses for a shocking number of platforms, but Maple is not on the list. Thankfully, the folks at Maple are working on that, and in the meantime we can use their fork to make opencode run on Maple, with any model Maple offers.

After tinkering to get Maple’s proxy service into opencode as a provider, I realized the Maple team had already done the work.

Anything that comes after bun run dev will be passed in as a command line argument. So bun run dev auth login gets parsed as opencode auth login, which is the proper command to trigger the login mechanism. Since opencode /directory/ is the correct syntax to run opencode in a specific folder, we’ll use that when running the full program:

Step 5: Evaporate your Tokens

Using these tools consumes significantly more tokens than a chat session, especially with lots of files, models that spawn agents, or models that do lots of reasoning. Whether the cost tradeoff makes sense is based entirely on what your time is worth. Happy hacking!
